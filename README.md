# XV6 文件系统优化与扩展实验

## 项目概述

本实验基于 xV6 操作系统，旨在对其文件系统模块进行优化和扩展。通过引入更复杂的算法和设计，优化文件系统的性能、可靠性和并发性，增加更多高级功能，如目录管理、文件权限控制、符号链接、挂载多文件系统等。实验还重点对文件系统的缓存机制进行优化，以提高磁盘 I/O 性能。

## 项目目标

1. **扩展文件系统功能**：
   - 增加 **目录** 管理功能，支持目录的创建、删除和遍历。
   - 增加 **文件权限控制**，支持文件的读、写、执行权限的设置和检查。
   - 支持 **符号链接**，即创建指向其他文件或目录的软链接。
   - 实现 **多文件系统挂载**，允许挂载不同类型的文件系统。
2. **优化文件系统缓存机制**：
   - 改进现有的文件系统缓存，减少磁盘 I/O 操作，提高文件读取和写入的效率。
   - 引入 **LRU（最少使用）缓存策略**，提高缓存命中率。
3. **提高磁盘 I/O 性能**：
   - 改进磁盘调度算法，减少寻道时间，优化磁盘 I/O 操作。
4. **增强并发处理能力**：
   - 改进文件锁机制，支持更高并发的文件访问。
5. **实现容错与崩溃恢复机制**：
   - 在文件系统操作过程中增加日志记录功能，确保系统崩溃时能够恢复到一致性状态。

## 技术栈

- **操作系统**：xV6（基于 Unix 的教学操作系统）
- **编程语言**：C
- **文件系统类型**：xV6 默认文件系统（基于 inode 和块的结构）
- **算法**：LRU 缓存算法、B+ 树索引、磁盘调度算法（LOOK, C-SCAN）
- **并发控制**：信号量、互斥锁、条件变量等

## 核心功能与模块

### 1. 扩展文件系统功能

- **目录管理**：
  - 添加对目录的创建、删除和遍历操作的支持。支持嵌套目录结构。
- **文件权限控制**：
  - 增加文件权限（读、写、执行）的设置与检查，支持用户、组、其他用户三种权限模型。
- **符号链接（Symlink）**：
  - 允许创建符号链接（软链接）指向其他文件或目录，实现跨文件系统的引用。
- **多文件系统挂载**：
  - 实现挂载不同类型的文件系统支持，可以在一个操作系统环境中挂载多个文件系统（例如，ext4、FAT、NTFS等）。

### 2. 文件系统缓存优化

- **缓存机制**：
  - 使用 **LRU**（最少使用）算法来管理文件系统缓存，提高缓存的命中率，减少不必要的磁盘访问。
  - 引入缓存大小动态调整机制，提升系统性能。
- **缓存同步与失效**：
  - 设计缓存失效机制，确保数据的准确性和一致性，避免脏数据影响文件系统操作。

### 3. 磁盘 I/O 调度优化

- 磁盘调度算法
  - 使用 **LOOK** 或 **C-SCAN** 算法来优化磁盘的寻道时间，提升文件系统的响应速度。

### 4. 并发控制与容错机制

- 并发控制
  - 通过精细化的锁机制，允许多个进程并发操作文件，避免竞争条件（race condition）和死锁问题。
- 崩溃恢复
  - 在文件系统操作过程中，记录日志以便在系统崩溃时恢复文件系统状态，确保数据的完整性。

2. **编译 xV6 操作系统**：

   ```

   ```

3. **运行并测试优化后的文件系统**：

   ```
   make qemu
   ```

4. **测试扩展功能**：

   - 通过命令行或编写测试程序来验证目录操作、权限控制、符号链接以及多文件系统挂载等功能。

5. **测试缓存优化**：

   - 使用磁盘 I/O 性能测试工具，比较优化前后的性能差异，验证缓存机制改进带来的 I/O 提升。
